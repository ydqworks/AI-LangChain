## 10 行笔记

1. LangChain 技术栈分三层：应用层、Build 开源栈、LangSmith 云端平台。
2. 应用层是我的业务代码，负责入口和用户交互，调用下层 Agent。
3. Build 栈在本地 Node.js 运行，自下而上：core → langgraph → langchain → deepagents。
4. `@langchain/core` 是最底层共享抽象（消息、工具、Runnable）。
5. LangChain 适合简单 Agent；LangGraph 适合要精细控制流程的场景。
6. Deep Agents 适合复杂长时任务，内置规划、子 Agent、文件系统、记忆。
7. LangSmith 不改变 Agent 逻辑，负责 Monitor / Test / Deploy。
8. `LANGSMITH_TRACING=true` 可让 Build 栈自动上报 trace 到 LangSmith。
9. 学习顺序：先 Monitor 会看 trace，再逐层写 Build 代码。
10. API Key 存 `.env`，勿提交 Git。
